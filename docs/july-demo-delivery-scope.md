# 2026-07-10 Q1 Demo 交付功能列表

本文档用于确认 2026 年 7 月 10 日 Q1 Demo 展示范围。目标是把“AI 问答助手”拆成可验收的功能点，并区分必做、加分和暂缓。

## 1. Demo 目标

7 月 Demo 要展示一个可跑通的 RAG 问答闭环：

```text
用户输入问题
  -> 系统检索知识库
  -> 系统生成回答
  -> 页面展示回答
  -> 页面展示 citation / 来源
  -> 异常时给出明确提示
```

演示重点不是功能数量，而是主链路稳定、引用可信、异常可控。

## 2. Demo 必做功能

### 2.1 Web 页面基础能力

| 功能 | 验收标准 | 责任边界 |
|---|---|---|
| 页面正常加载 | 打开页面后能看到标题、Demo 提示、聊天区、输入框、发送按钮 | Vue |
| 输入问题 | 用户能输入自然语言问题 | Vue |
| 发送问题 | 支持点击按钮和 Enter 发送 | Vue |
| 多行输入 | Shift + Enter 换行，不触发发送 | Vue |
| 空输入限制 | 空字符串和纯空格不发送，并提示请输入问题 | Vue |
| 防重复提交 | 请求进行中不会重复触发多个请求 | Vue |
| 消息展示 | 用户问题和助手回答区分展示 | Vue |
| loading 状态 | 生成过程中有明确 loading / generating 状态 | Vue |
| 请求结束恢复 | 成功或失败后输入框恢复可用 | Vue |

### 2.2 Web 后端 / BFF 能力

| 功能 | 验收标准 | 技术建议 |
|---|---|---|
| Web API | 提供 `/api/v1/chat` 给 Vue 调用 | Java Spring Boot |
| 请求校验 | 缺少 query 或 query 为空时返回稳定错误 | Spring Validation |
| trace_id | 每次请求都能生成或透传 trace_id | Filter / Interceptor |
| Mock 模式 | Agent 未完成时仍可演示主流程 | Spring Profile / 配置开关 |
| Agent 转发 | 能把 Web 请求转发给 Agent 层 | WebClient |
| 统一响应 | 返回 `status`、`answer`、`trace_id`、`citations` | DTO |
| 异常映射 | Agent、网络、超时等异常映射为 Web 状态 | ControllerAdvice |
| 日志 | 根据 trace_id 查询一次请求链路 | 结构化日志 |

### 2.3 真实问答链路

| 功能 | 验收标准 | 责任边界 |
|---|---|---|
| 调用 Agent | Vue 通过 BFF 调用真实 Agent，而不是直连 Agent | Vue + BFF |
| 知识库检索 | 输入固定 Demo 问题后，Agent 能检索到相关文档 | Agent + 工具集 |
| 答案生成 | Agent 返回自然语言答案 | Agent |
| 完整答案展示 | 不支持流式时，页面也能展示完整答案 | Vue + BFF |
| 流式输出 | 支持时答案逐步显示，结束后 loading 消失 | Vue + BFF + Agent |
| 不暴露原始 SSE | 用户只看到答案，不看到 `data:` 原始内容 | Vue |

### 2.4 Citation / 来源展示

| 功能 | 验收标准 | 责任边界 |
|---|---|---|
| 引用编号 | 每条引用有编号，如 `[1]` | Vue |
| 引用标题 | 展示来源文档标题 | Vue + Agent |
| 链接展示 | `source_url` 存在时可点击打开 | Vue + 数据层 |
| 无链接引用 | `source_url` 为空时不生成无效链接 | Vue |
| 文档标识 | 展示或记录 `doc_id`、`chunk_id` | Vue + Agent |
| 引用与答案一起返回 | 回答完成后能看到 citations | Agent + BFF |

### 2.5 文档入库能力

Demo 必须保证至少一种“文档进入知识库”的方式可用。

| 功能 | Q1 要求 | 推荐做法 |
|---|---|---|
| 演示文档准备 | 必做 | 2026-07-10 前预置入库 |
| 文档解析 | 必做但不一定在 Web 页面展示 | Java 数据处理服务 + Apache Tika |
| 文档分块 | 必做 | 后端数据处理管线 |
| 向量化 | 必做 | 后端数据处理管线 |
| 写入存储 | 必做 | 向量存储、文档存储、元数据存储 |
| 页面上传文档 | 加分 | 时间允许再做 |
| 导入状态展示 | 加分 | 若做上传，则需要任务状态接口 |

如果时间紧，7 月 Demo 可以采用“预置文档已经入库 + 页面问答展示引用”的方式。这样仍然满足 RAG 闭环，但降低现场上传失败风险。

### 2.6 异常提示

| 场景 | 页面提示 | 验收标准 |
|---|---|---|
| 空输入 | 请输入问题 | 不发送请求 |
| 无相关文档 | 当前知识库没有足够信息回答该问题 | 页面不白屏，输入恢复 |
| 检索异常 | 检索服务暂时不可用，请稍后重试 | loading 结束，保留 trace_id |
| 模型异常 | 模型服务暂时不可用，请稍后重试 | loading 结束，保留 trace_id |
| 网络异常 | 网络连接异常，请检查服务是否启动 | 可继续提问 |
| 请求超时 | 请求超时，请稍后重试 | 可继续提问 |
| 流式中断 | 生成中断，请稍后重试 | 不展示残缺 SSE 原文 |

### 2.7 基础操作

| 功能 | 验收标准 |
|---|---|
| 复制回答 | 点击复制后，答案正文进入剪贴板 |
| 清空对话 | 点击清空后，聊天区恢复初始或空状态 |
| trace_id 展示 | 每次助手响应能看到或记录 trace_id |
| Mock 兜底 | 真实 Agent 不稳定时可以切换 Mock 演示 |

## 3. 加分功能

这些功能有价值，但不应阻塞 2026-07-10 Q1 Demo。

| 功能 | 价值 | 前置依赖 |
|---|---|---|
| Web 上传文档 | 现场展示文档入库更完整 | 文件上传、Tika 入库接口、任务状态 |
| 文档导入状态页 | 用户知道解析、分块、向量化进度 | 数据处理服务状态接口 |
| 用户反馈 | 收集答案质量，如赞、踩、纠错 | 反馈接口和存储 |
| 历史会话 | 支持查看之前的问题 | 会话存储 |
| 登录和权限 | 控制文档可见范围 | 鉴权系统、权限过滤 |
| 日志查询页面 | 方便根据 trace_id 排查 | 日志服务或查询接口 |
| 多轮对话 | 提升问答体验 | Agent 上下文能力 |
| 多 Agent 协作展示 | 展示复杂推理流程 | Agent 编排能力 |
| 文档预览和片段定位 | citation 更可信 | 文档服务、片段定位 |

## 4. 暂缓功能

以下功能复杂度较高，建议 Q1 不做或只预留接口。

- 完整用户权限体系；
- 多租户文档隔离；
- 企业级审计；
- 长期历史会话管理；
- 文档版本管理；
- 大规模并发压测；
- 复杂 Rerank 策略可视化；
- 多 Agent 任务编排；
- 答案人工审核流；
- 引用定位到 PDF 页码高亮。

## 5. Demo 推荐脚本

### 5.1 正常问答

输入：

```text
项目 Q1 阶段需要完成哪些功能？
```

预期：

- 页面展示用户问题；
- loading 出现；
- 答案逐步输出或完整展示；
- 展示 `trace_id`；
- 展示至少一条 citation；
- citation 包含标题、`doc_id`、`chunk_id`，有链接则可点击。

### 5.2 无相关文档

输入：

```text
请回答一个知识库外的问题
```

预期：

- 页面提示“当前知识库没有足够信息回答该问题”；
- loading 结束；
- 输入框可继续使用。

### 5.3 异常场景

通过 Mock 或后端参数模拟：

- `retrieval_error`
- `llm_error`
- `network_error`
- `timeout_error`
- `stream_error`

预期：

- 页面展示对应错误提示；
- 不白屏；
- 不死锁；
- 保留 trace_id；
- 用户可以继续提问。

### 5.4 引用无链接

让 Agent 或 Mock 返回：

```json
{
  "index": 1,
  "title": "本地 PDF 解析结果",
  "doc_id": "local-policy-pdf",
  "chunk_id": "chunk-021",
  "source_url": ""
}
```

预期：

- 页面展示标题、`doc_id`、`chunk_id`；
- 不生成空链接或错误链接。

## 8. 结论

2026-07-10 Q1 Demo 的最低成功标准是：

```text
用户能在 Web 页面提问；
系统能基于已入库文档生成回答；
回答能展示来源引用；
异常时页面能给出明确提示；
真实 Agent 不稳定时可用 Mock 兜底完成演示。
```

Web 层必须交付 Vue 前端和 Java Spring Boot BFF。文档入库、检索和 LLM 由其他层完成，但 Web 层需要提前拿到接口契约、演示文档清单和测试问题，确保最终链路可以在 2026-07-10 前稳定跑通。
